### Notes

- `assets/js/life-money/*` is committed source code (not generated by Jest).
- `jest.global-setup.js` ensures `assets/js/life-money/` exists before tests run.
- `jest.setup.js` is a lightweight `setupFilesAfterEnv` hook and does not generate files.
- Unit tests import committed modules using:
  - `require('../../../../assets/js/life-money/compute.js')`
  - `require('../../../../assets/js/life-money/chart.js')`

### Test flow

1. Run `npm test`
2. Jest executes `globalSetup` (`jest.global-setup.js`)
3. Jest executes `setupFilesAfterEnv` (`jest.setup.js`)
4. Jest runs all discovered test suites, including:
   - `test/unit/assets/js/compute.test.js`
   - `test/unit/assets/js/chart.test.js`
